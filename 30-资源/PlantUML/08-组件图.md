# 08 · 组件图（Component）

← [[07-状态图]] · [[PlantUML从入门到精通|目录]] · 下一章 → [[09-部署图]]

官方：https://plantuml.com/zh/component-diagram

组件图看**软件如何拆成可部署/可替换的块**及依赖。画微服务边界、前后端分层时很常用。

---

## 1. 组件与接口

```plantuml
@startuml
[第一组件]
[另一个组件] as Comp2
component Comp3
component [最后组件] as Comp4

() "支付接口" as PayAPI
[订单服务] --> PayAPI
PayAPI --> [支付服务]
@enduml
```

`[名称]` 或 `component`；`()` 表示接口。

---

## 2. 包分层与数据库

```plantuml
@startuml
title 简洁分层

package "前端" {
  [Web]
  [App]
}

package "网关" {
  [API Gateway]
}

package "服务" {
  [用户服务]
  [订单服务]
}

database "MySQL" {
  [用户库]
  [订单库]
}

[Web] --> [API Gateway]
[App] --> [API Gateway]
[API Gateway] --> [用户服务]
[API Gateway] --> [订单服务]
[用户服务] --> [用户库]
[订单服务] --> [订单库]
@enduml
```

---

## 3. 完整样例：微服务草图

```plantuml
@startuml
title 微服务组件草图

package "接入" {
  [Web] as Web
  [Mobile] as Mobile
  [API Gateway] as GW
}

package "业务服务" {
  [用户服务] as User
  [订单服务] as Order
  [商品服务] as Product
  [支付服务] as Pay
}

package "基础设施" {
  [认证中心] as Auth
  [注册中心] as Reg
}

cloud "第三方" {
  [微信支付]
}

database DB {
  [用户库]
  [订单库]
  [商品库]
}

Web --> GW
Mobile --> GW
GW --> User
GW --> Order
GW --> Product
Order --> Pay
Pay --> [微信支付]
User --> Auth
Order --> Auth
User --> Reg
Order --> Reg
Product --> Reg
Pay --> Reg
User --> [用户库]
Order --> [订单库]
Product --> [商品库]
@enduml
```

---

## 4. 与部署图的分工

| 问题 | 用 |
|------|-----|
| 逻辑模块谁依赖谁 | **组件图** |
| 跑在哪台机器 / 哪个 K8s 服务 | **部署图** |

可先组件、后部署；或同主题各一张。

---

## 5. 练习

1. 画出你当前项目真实的「前端 / BFF / 两服务 / 一库」。  
2. 标出一个外部依赖（对象存储或支付），放入 `cloud`。  
3. 给最容易出问题的依赖加 `note`。
