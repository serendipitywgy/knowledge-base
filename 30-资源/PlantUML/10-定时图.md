# 10 · 定时图（Timing）

← [[09-部署图]] · [[PlantUML从入门到精通|目录]] · 下一章 → [[11-非UML常用图]]

官方：https://plantuml.com/zh/timing-diagram

> 注意：官网中文页有时把 Timing 也译成「时序图」，易与 Sequence 混淆。  
> **Sequence = 消息顺序（[[02-时序图]]）；Timing = 时间轴上的状态/信号。**

定时图适合：超时、缓存新鲜度、握手时序、硬件/协议波形。

---

## 1. 参与者类型

| 关键字 | 含义 |
|--------|------|
| `concise` | 紧凑状态条（适合业务状态） |
| `robust` | 折线状态（状态迁移更醒目） |
| `binary` | 高低电平二值信号 |
| `clock` | 时钟（`period` / `pulse` / `offset`） |
| `analog` | 连续模拟量（线性插值） |

用 `@时刻` + `is 状态` 描述变化；可用消息 `A -> B : 文案`。

---

## 2. 最小例子（Web 请求）

```plantuml
@startuml
robust "Web 浏览器" as WB
concise "Web 用户" as WU

@0
WU is 空闲
WB is 空闲

@100
WU -> WB : URL
WU is 等待中
WB is 处理中

@300
WB is 等待中
@enduml
```

---

## 3. 相对时间与按参与者书写

```plantuml
@startuml
robust "DNS" as DNS
robust "Browser" as WB
concise "User" as WU

@0
WU is Idle
WB is Idle
DNS is Idle

@+100
WU -> WB : URL
WU is Waiting
WB is Processing

@+200
WB is Waiting
WB -> DNS@+50 : Resolve

@+100
DNS is Processing

@+300
DNS is Idle
@enduml
```

也可把同一参与者的变化写在一块：

```plantuml
@startuml
robust "Browser" as WB
concise "User" as WU

@WB
0 is idle
+200 is Processing
+100 is Waiting

@WU
0 is Waiting
+500 is ok
@enduml
```

---

## 4. 时钟、二进制、模拟量

```plantuml
@startuml
clock clk with period 50
binary "Enable" as EN
analog "Voltage" as V

@0
EN is low
V is 0

@100
EN is high
V is 3

@300
EN is low
V is 1
@enduml
```

`clock "C1" as C1 with period 50 pulse 15 offset 10` 可调脉宽与偏移。

---

## 5. 锚点、约束、高亮

```plantuml
@startuml
binary "enable" as EN
concise "dataBus" as db

@0 as :start
@5 as :en_high
@10 as :en_low

@:start
EN is low
db is "0x0000"

@:en_high
EN is high

@:en_high-2
db is "0xf23a"

@:en_low
EN is low

db@:en_high-1 <-> @:en_low : setup time
highlight :en_high to :en_low #Gold : Write window
@enduml
```

相对锚点：`@:en_high+6`、小数时刻 `@5.5`。  
时间约束：`WB@0 <-> @50 : {50 ms lag}`。

---

## 6. 缩放、日期轴、隐藏时间轴

```plantuml
@startuml
concise "User" as WU
scale 100 as 50 pixels
@WU
0 is Waiting
+500 is ok
@enduml
```

日期：

```plantuml
@startuml
robust "Browser" as WB
concise "User" as WU
use date format "YY-MM-dd"

@2019/07/02
WU is Idle
WB is Idle

@2019/07/04
WU is Waiting
WB is Processing
@enduml
```

`hide time-axis` 可隐藏轴；`manual time-axis` 仅在状态变化处打标。

## 7. 完整样例：HTTP 缓存新鲜度

```plantuml
@startuml
title Client / Server / Cache freshness
concise "Client" as Client
concise "Server" as Server
concise "Cache" as Cache

Server is idle
Client is idle

@Client
0 is send
Client -> Server@+25 : GET
+25 is await
+75 is recv
+25 is idle
+25 is send
Client -> Server@+25 : GET\nIf-Modified-Since
+25 is await
+50 is recv
+25 is idle

@100 <-> @275 : 无需回源

@Server
25 is recv
+25 is work
+25 is send
Server -> Client@+25 : 200 Expires:275
+25 is idle
+75 is recv
+25 is send
Server -> Client@+25 : 304
+25 is idle

@Cache
75 is fresh
+200 is stale
@enduml
```

## 8. 与 Sequence 对照

| 问题 | Sequence | Timing |
|------|----------|--------|
| 谁调用谁、参数 | ✅ | 弱 |
| 状态持续多久、是否重叠 | 弱 | ✅ |
| API 文档主图 | ✅ | 补充超时/SLA |

同一主题可两张图互相链接。

---

## 9. 练习

1. 画「200ms 超时」：User Waiting 与 API Processing 的重叠关系。  
2. 用 `highlight` 标出超时危险区间。  
3. 用 `binary` + `clock` 示意一次片选/使能脉冲（不一定真实硬件）。
