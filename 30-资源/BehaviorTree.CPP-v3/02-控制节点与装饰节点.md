## 1. 选型总表（先记这个）

| 需求 | 优先节点 | 备注 |
|------|----------|------|
| 步骤 A→B→C，一步失败整体失败 | `Sequence` | 最常用 |
| 记住进度，下次 tick 从上次继续 | `SequenceStar` | 带 memory |
| 每 tick 都从头检查条件 | `ReactiveSequence` | 条件变化要立刻反应 |
| 尝试 A，失败再试 B | `Fallback`（Selector） | 恢复策略主干 |
| 带 memory 的 Fallback | `FallbackStar` | |
| 条件变化时重选分支 | `ReactiveFallback` | |
| 多子节点同时推进 | `Parallel` | 注意成功/失败阈值 |
| 失败重试 N 次 | `RetryUntilSuccessful` | Decorator |
| 重复 N 次 | `Repeat` | |
| 限时 | `Timeout` | 超时 → FAILURE |
| 取反 | `Inverter` | |
| 强制成功/失败 | `ForceSuccess` / `ForceFailure` | 慎用，易掩盖错误 |

v3 文档里 Fallback ≈ 其他库的 Selector。

## 2. Sequence：顺序成功

从左到右。子节点 SUCCESS → 下一个；FAILURE → 整段 FAILURE；RUNNING → 整段 RUNNING。

```xml
<Sequence name="navigate">
  <IsLocalized/>
  <ComputePathToPose goal="{goal}" path="{path}"/>
  <FollowPath path="{path}"/>
</Sequence>
```

**工程坑**：`Sequence`（无 Star）在子节点 SUCCESS 后，**下次从根 tick 可能重置**，不适合「跨多次外部事件记住进度」的长流程。需要记忆用 `SequenceStar`，或把状态放进黑板由上层任务机推进。

## 3. Fallback：失败才换路

子节点 FAILURE → 试下一个；任一 SUCCESS → 整段 SUCCESS。

```xml
<Fallback name="nav_or_recover">
  <Sequence name="primary">
    <ComputePathToPose goal="{goal}" path="{path}"/>
    <FollowPath path="{path}"/>
  </Sequence>
  <Sequence name="recover">
    <ClearEntireCostmap service_name="local_costmap/clear_entirely_local_costmap"/>
    <Spin/>
  </Sequence>
</Fallback>
```

这是 Nav2「导航失败 → 清图/自旋」的骨架。

## 4. Reactive*：每 tick 重新评估

`ReactiveSequence` / `ReactiveFallback` 会在每次 tick **重新从第一个子节点**评估（已成功的也可能被重置语义影响）。

适用：

- 「电池过低立刻打断当前任务」  
- 「人靠近立刻切换到避让」

不适用：昂贵的一次性初始化放在 Reactive 左侧（会被反复执行）。把重初始化放到 `onStart` 或非 Reactive 子树。

## 5. Parallel：并行

配置「需要多少子节点 SUCCESS / FAILURE」才结束。典型：一边跟路径一边监控碰撞 / 电池。

注意：

- 并行子树共享黑板，**写同一 key 要有约定**  
- 一方失败时，是否 halt 另一方必须在节点/树上明确  

## 6. 装饰节点（Decorator）

只包装**一个**子节点。

```xml
<RetryUntilSuccessful num_attempts="3">
  <ComputePathToPose goal="{goal}" path="{path}"/>
</RetryUntilSuccessful>

<Timeout msec="5000">
  <WaitForDoorOpen/>
</Timeout>

<Inverter>
  <IsBatteryLow/>
</Inverter>
```

| Decorator | 工程用途 |
|-----------|----------|
| `RetryUntilSuccessful` | 规划偶发失败、传感器抖动 |
| `Timeout` | 等待门开、等待定位，防死等 |
| `RateController`（若发行版提供） | 降频 tick 重规划 |
| `KeepRunningUntilFailure` | 持续监控类条件 |

**反模式**：用 `ForceSuccess` 吞掉导航失败却不打日志——线上难查。

## 7. 组合套路（可直接套）

### 7.1 检查 → 执行

```xml
<Sequence>
  <ConditionA/>
  <ActionB/>
</Sequence>
```

### 7.2 主路径 / 恢复

```xml
<Fallback>
  <PrimaryTask/>
  <RecoveryTask/>
</Fallback>
```

### 7.3 限时重试

```xml
<Timeout msec="30000">
  <RetryUntilSuccessful num_attempts="5">
    <DockToStation/>
  </RetryUntilSuccessful>
</Timeout>
```
