## 1. 推荐包布局

```text
my_robot_bt/
  package.xml
  CMakeLists.txt
  include/my_robot_bt/
    nodes/
      navigate_to_pose_node.hpp
      set_waypoint_node.hpp
      is_battery_ok.hpp
    mission_bt_node.hpp          # 跑树的 ROS 节点
  src/
    nodes/
      navigate_to_pose_node.cpp
      ...
    plugins.cpp                  # BT_REGISTER_NODES
    mission_bt_node.cpp
  trees/
    patrol.xml
    subtree_navigate.xml
    subtree_recover.xml
  config/
    mission_bt.yaml
  test/
    test_sequence_logic.cpp
    test_tree_load.cpp
    trees/test_patrol.xml
  launch/
    mission_bt.launch.py
```

原则：**节点代码 / 树 XML / 参数 / 测试** 分离。

## 2. CMake 要点

```cmake
find_package(ament_cmake REQUIRED)
find_package(rclcpp REQUIRED)
find_package(behaviortree_cpp_v3 REQUIRED)
find_package(nav2_msgs REQUIRED)
# find_package(nav2_behavior_tree REQUIRED)  # 若复用 BtActionNode

add_library(my_robot_bt_nodes SHARED
  src/nodes/navigate_to_pose_node.cpp
  src/plugins.cpp)
ament_target_dependencies(my_robot_bt_nodes
  rclcpp behaviortree_cpp_v3 nav2_msgs)

add_executable(mission_bt_node src/mission_bt_node.cpp)
target_link_libraries(mission_bt_node my_robot_bt_nodes)
ament_target_dependencies(mission_bt_node rclcpp behaviortree_cpp_v3)

install(TARGETS my_robot_bt_nodes mission_bt_node
  ARCHIVE DESTINATION lib
  LIBRARY DESTINATION lib
  RUNTIME DESTINATION lib/${PROJECT_NAME})
install(DIRECTORY trees config launch
  DESTINATION share/${PROJECT_NAME})
```

## 3. 配置驱动，而不是硬编码树

```cpp
std::string xml_file = declare_parameter<std::string>("tree_xml_file", "");
auto plugin_libs = declare_parameter<std::vector<std::string>>("plugin_lib_names", {});
for (const auto& lib : plugin_libs) {
  factory.registerFromPlugin(lib);  // 或按 Nav2 方式拼 so 名
}
auto tree = factory.createTreeFromFile(xml_file, blackboard);
```

现场改策略 → 改 `trees/*.xml` + 重启/重载；少出二进制。

## 4. 可测试性分层

| 层 | 测什么 | 怎么测 |
|----|--------|--------|
| 纯节点 | Port 校验、状态迁移 | GTest，mock 依赖 |
| XML 装载 | 能 createTree、端口类型通过 | `createTreeFromFile` |
| 控制流 | Fallback/Retry 行为 | 用 `AlwaysSuccess`/`AlwaysFailure` 替身节点 |
| 集成 | 对上 Nav2 Action | launch test / 仿真 |

替身节点示例：

```cpp
factory.registerSimpleAction("FakeNavigate",
  [&](BT::TreeNode& n) {
    // 读一个测试用黑板开关决定 SUCCESS/FAILURE
    return BT::NodeStatus::SUCCESS;
  });
```

单测里加载「测试专用 XML」，不要依赖真机。

## 5. 日志与可观测性

- 节点内：`RCLCPP_INFO(logger, "[%s] ...", name().c_str());`  
- 树级：`BT::StdCoutLogger` / `FileLogger`（v3 提供的 logger）挂到 `tree`  
- 线上：把关键状态（当前叶子、最后 error_code）发布到 topic，便于 HMI
