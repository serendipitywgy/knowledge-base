## 1. 选基类

| 基类 | 何时用 |
|------|--------|
| `BT::SyncActionNode` | 一次 tick 内完成（写黑板、轻量 IO） |
| `BT::ConditionNode` | 纯判断，返回 SUCCESS/FAILURE |
| `BT::StatefulActionNode` | **推荐**：长任务，分 `onStart`/`onRunning`/`onHalted` |
| `BT::AsyncActionNode` | 必须丢到别的线程跑的阻塞库 |

## 2. StatefulActionNode 模板（工程默认）

```cpp
#include "behaviortree_cpp_v3/action_node.h"

class DockToStation : public BT::StatefulActionNode
{
public:
  DockToStation(const std::string& name, const BT::NodeConfiguration& config)
  : BT::StatefulActionNode(name, config) {}

  static BT::PortsList providedPorts()
  {
    return {
      BT::InputPort<std::string>("station_id"),
      BT::OutputPort<int>("error_code"),
    };
  }

  BT::NodeStatus onStart() override
  {
    if (!getInput("station_id", station_id_)) {
      return BT::NodeStatus::FAILURE;
    }
    // 发送目标 / 启动状态机
    started_ = true;
    return BT::NodeStatus::RUNNING;
  }

  BT::NodeStatus onRunning() override
  {
    // 查询进度；完成 SUCCESS；失败 FAILURE；否则 RUNNING
    if (done_ok_) return BT::NodeStatus::SUCCESS;
    if (done_fail_) {
      setOutput("error_code", code_);
      return BT::NodeStatus::FAILURE;
    }
    return BT::NodeStatus::RUNNING;
  }

  void onHalted() override
  {
    // 取消底层动作、清定时器；可被父节点打断时必调
    cancelUnderlying();
  }

private:
  std::string station_id_;
  bool started_{false};
  bool done_ok_{false};
  bool done_fail_{false};
  int code_{0};
};
```

规则：

- **禁止**在 `onRunning` 里长时间阻塞  
- `onHalted` 必须幂等、可重入  
- 进度靠成员变量或黑板，不要依赖「静态局部」

## 3. 注册到 Factory

```cpp
void registerMyNodes(BT::BehaviorTreeFactory& factory)
{
  factory.registerNodeType<DockToStation>("DockToStation");
  factory.registerSimpleCondition("IsBatteryOk",
    [](BT::TreeNode&) {
      return battery_ok() ? BT::NodeStatus::SUCCESS
                          : BT::NodeStatus::FAILURE;
    });
}
```

XML 标签名 = 注册名：`<DockToStation station_id="A1"/>`。

## 4. 插件化（`.so`）

多团队 / 多车型时，把节点打成插件，主程序只 `registerFromPlugin`。

```cpp
// dock_nodes_plugin.cpp
#include "behaviortree_cpp_v3/bt_factory.h"

BT_REGISTER_NODES(factory)
{
  factory.registerNodeType<DockToStation>("DockToStation");
}
```

```cmake
add_library(dock_bt_nodes SHARED src/dock_nodes_plugin.cpp)
target_link_libraries(dock_bt_nodes behaviortree_cpp_v3 /* ... */)
# 安装到 lib/，启动参数传入 plugin 路径列表
```

```cpp
factory.registerFromPlugin("/path/to/libdock_bt_nodes.so");
```

Nav2 即用插件列表参数加载 BT 节点。

## 5. 节点设计清单

- [ ] `providedPorts()` 完整，含默认值与注释字符串  
- [ ] 输入缺失 → FAILURE + 日志  
- [ ] 长任务可取消（`onHalted`）  
- [ ] 不在 tick 路径抛未捕获异常（捕获后转 FAILURE）  
- [ ] 不持有悬空 Node / Executor 指针  
- [ ] 单测：成功路径、失败路径、中途 halt  

## 6. 反模式

| 反模式 | 后果 | 改法 |
|--------|------|------|
| Sync 里 spin Action | 卡死整树 | Stateful + 异步 client |
| 节点内 `sleep` 代替 Timeout | 无响应取消 | Decorator `Timeout` 或检查时钟 |
| 未声明 port 直接搓黑板 | 隐式耦合 | Ports + remapping |
| 一个节点做完巡检全流程 | 无法复用 | 拆小 Action + XML 编排 |
