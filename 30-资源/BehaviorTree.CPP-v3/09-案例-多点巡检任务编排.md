# 09 · 案例：多点巡检任务编排

## 1. 需求

AMR 按列表巡检：`wp1 → 作业 → wp2 → 作业 → …`，要求：

- 导航复用 Nav2（`NavigateToPose`）  
- 单点失败可跳过或中止（可配置）  
- 电池低则中断去充电点  
- 任务可取消；支持从第 N 个点恢复  

这是上层**任务 BT** 的典型形态。

## 2. 包内角色划分

| 组件 | 职责 |
|------|------|
| `mission_bt_node` | 加载 XML、tick、接任务话题 |
| `NavigateToPose` 节点 | Action client → Nav2 |
| `DoInspection` 节点 | 拍照 / 上报 / 等业务 |
| `IsBatteryLow` | Condition |
| `PopNextWaypoint` | 从队列取点写入 `{goal}` |
| `trees/patrol.xml` | 编排 |

## 3. 主树 XML

```xml
<root main_tree_to_execute="PatrolMission">
  <BehaviorTree ID="InspectOne">
    <Sequence name="inspect_one">
      <NavigateToPose goal="{goal}" error_code="{nav_error}"/>
      <DoInspection station_id="{station_id}" report="{report}"/>
    </Sequence>
  </BehaviorTree>

  <BehaviorTree ID="PatrolMission">
    <ReactiveFallback name="battery_or_work">
      <Sequence name="go_charge">
        <IsBatteryLow threshold="20.0"/>
        <NavigateToPose goal="{charge_pose}"/>
        <Wait msec="1000"/>
      </Sequence>

      <Sequence name="work_loop">
        <RetryUntilSuccessful num_attempts="2">
          <Sequence>
            <PopNextWaypoint goal="{goal}" station_id="{station_id}" queue="{wp_queue}"/>
            <Fallback name="skip_or_fail">
              <SubTree ID="InspectOne"/>
              <Sequence name="skip_point">
                <LogWarning message="skip waypoint"/>
                <AlwaysSuccess/>
              </Sequence>
            </Fallback>
          </Sequence>
        </RetryUntilSuccessful>

        <!-- 简化：真实项目用「While queue not empty」自定义 Decorator / 循环节点 -->
        <IsQueueEmpty queue="{wp_queue}"/>
      </Sequence>
    </ReactiveFallback>
  </BehaviorTree>
</root>
```

说明：

- `ReactiveFallback` + `IsBatteryLow`：每 tick 可抢占去充电  
- `Fallback` + 跳过：单点导航失败不拖死整轮（按产品改成 `AlwaysFailure` 即「失败即停」）  
- 队列循环：v3 无通用 While 时，用自定义 `WhileQueueNotEmpty` Decorator，或在 ROS 节点里「外层 while + 每轮 create/tick 子树」

## 4. 关键自定义节点接口

### 4.1 PopNextWaypoint

```cpp
static BT::PortsList providedPorts() {
  return {
    // v3：读写同一黑板 key 时，用同名 Input+Output，或节点内锁定后改
    BT::InputPort<std::vector<Waypoint>>("queue"),
    BT::OutputPort<std::vector<Waypoint>>("queue"),
    BT::OutputPort<geometry_msgs::msg::PoseStamped>("goal"),
    BT::OutputPort<std::string>("station_id"),
  };
}
// tick: 空队列 → FAILURE；否则 pop，setOutput("queue", rest) 并写 goal
```

### 4.2 DoInspection

`StatefulActionNode`：`onStart` 触发业务，`onRunning` 等结果，超时用外层 `Timeout`。

### 4.3 IsBatteryLow

`ConditionNode`：订阅 `/battery_state` 的缓存值，阈值判断。

## 5. 任务入口（工程伪代码）

```cpp
// 订阅 mission topic
void onMission(const MyMission& msg) {
  tree.haltTree();  // 取消当前
  auto bb = tree.rootBlackboard();
  bb->set("wp_queue", msg.waypoints);
  bb->set("charge_pose", charge_pose_);
  // 下一帧 tick 从根重新跑
}
```

取消：`tree.haltTree()`，确保所有 Stateful 节点 `onHalted` 取消 Navigate Action。

## 6. 从第 N 点恢复

持久化「已完成 station_id 列表」到文件 / 参数服务；派发任务前过滤 `wp_queue`。  
不要把进度只藏在 SequenceStar 内存里——进程重启会丢。

## 7. 仿真验收清单

- [ ] 三点巡检全成功  
- [ ] 第二点导航失败 → 跳过 → 第三点仍执行  
- [ ] 中途电池低于阈值 → 去充电（可先打桩 Condition）  
- [ ] 取消任务 → 底盘停止  
- [ ] 重发任务 → 新队列覆盖
