## 1. 行为树是什么（工程视角）

行为树（Behavior Tree, BT）把机器人决策拆成**可组合节点**：控制流在树里，动作在叶子节点。  
每帧（或每个周期）对根节点 `tick()` 一次，节点返回三种结果之一：

| 状态 | 含义 | 典型场景 |
|------|------|----------|
| `SUCCESS` | 完成 | 到达目标、检测通过 |
| `FAILURE` | 失败 | 规划失败、电池过低 |
| `RUNNING` | 进行中 | 跟路径、等待 Action 回执 |

和有限状态机（FSM）比：BT 更适合**层次任务 + 频繁插入恢复策略**；FSM 在状态爆炸时更难维护。

## 2. 为何选 BehaviorTree.CPP v3

| 能力 | 工程价值 |
|------|----------|
| XML 描述树形态 | 不改代码即可调策略；可进 Git / Code Review |
| Ports + Blackboard | 节点解耦，数据流显式 |
| 插件化自定义节点 | Action/Condition 可独立编译、按需加载 |
| ROS2 / Nav2 生态 | Humble 及多数产线栈仍以 `behaviortree_cpp_v3` 为主 |

**主战场**：导航恢复、多点巡检、上下料任务编排、替代臃肿 FSM。

## 3. v3 vs v4（选型别踩坑）

| 维度 | v3（本系列） | v4 |
|------|--------------|-----|
| ROS 包名 | `behaviortree_cpp_v3` | `behaviortree_cpp` |
| include | `behaviortree_cpp_v3/...` | `behaviortree_cpp/...` |
| XML | `main_tree_to_execute` | 常带 `BTCPP_format="4"` |
| 可视化 | **Groot**（一代） | **Groot2** |
| Nav2 | Humble / Iron 常用 v3 | Jazzy+ 逐步迁 v4 |

开项目前先确认：发行版依赖的是哪个包。混用 include / XML 方言是常见编译与加载失败原因。

## 4. 环境与依赖

### 4.1 ROS2 工作空间（推荐）

```bash
# 以 Humble 为例
sudo apt update
sudo apt install ros-humble-behaviortree-cpp-v3

# 若做导航任务编排，一并装 Nav2（按需）
sudo apt install ros-humble-navigation2 ros-humble-nav2-bringup
```

验证头文件：

```bash
dpkg -L ros-humble-behaviortree-cpp-v3 | grep bt_factory.h
```

### 4.2 纯 CMake（无 ROS）

从 [v3.8 分支](https://github.com/BehaviorTree/BehaviorTree.CPP/tree/v3.8) 构建；需要 C++14/17 编译器。产线机器人项目优先走发行版 apt / 工作空间，避免私有 fork 分叉。

## 5. 五分钟最小可运行

`package.xml` 依赖：`behaviortree_cpp_v3`。  
`CMakeLists.txt`：`find_package(behaviortree_cpp_v3 REQUIRED)`，并 `target_link_libraries(... BT::behaviortree_cpp_v3)`（具体 target 名以发行版 `*Config.cmake` 为准，Humble 常见为 `behaviortree_cpp_v3`）。

```cpp
#include "behaviortree_cpp_v3/bt_factory.h"

static const char* xml_text = R"(
<root main_tree_to_execute="MainTree">
  <BehaviorTree ID="MainTree">
    <Sequence name="root">
      <AlwaysSuccess name="ok"/>
    </Sequence>
  </BehaviorTree>
</root>
)";

int main()
{
  BT::BehaviorTreeFactory factory;
  auto tree = factory.createTreeFromText(xml_text);
  BT::NodeStatus status = BT::NodeStatus::RUNNING;
  while (status == BT::NodeStatus::RUNNING) {
    status = tree.tickRoot();
  }
  return status == BT::NodeStatus::SUCCESS ? 0 : 1;
}
```

能编译、能跑通，再进入节点与黑板。

## 6. 本系列工程约定

1. **树在 XML，逻辑在 C++ 节点** — 不要在 XML 里塞业务算法  
2. **Ports 显式声明** — 禁止节点内部到处 `blackboard->get` 隐式耦合（子树边界除外）  
3. **长任务返回 RUNNING** — 阻塞 `tick` 线程会卡死整棵树  
4. **恢复策略用 Fallback / Retry 表达** — 少写巨型 if-else
