## 1. 需求（可写进设计说明）

机器人收到单个 `PoseStamped` 目标：

1. 规划路径并跟踪  
2. 失败则清局部代价地图并自旋，再重试  
3. 总重试不超过 3 次  
4. 全程可取消  

这是 Nav2 默认树的**简化教学版**，便于自研任务树理解恢复骨架。

## 2. 树结构

```text
RetryUntilSuccessful (3)
 └─ Fallback
      ├─ Sequence（主路径）
      │    ├─ ComputePathToPose
      │    └─ FollowPath
      └─ Sequence（恢复，失败后走这里）
           ├─ ClearEntireCostmap（local）
           └─ Spin
```

语义：主路径失败 → 恢复 → Retry 再进主路径；恢复也失败则消耗一次 Retry。

## 3. XML（可放入 `trees/navigate_with_recovery.xml`）

节点名按你注册的插件为准；若直接改 Nav2 树，用 Nav2 自带 ID。

```xml
<root main_tree_to_execute="NavigateWithRecovery">
  <BehaviorTree ID="NavigateWithRecovery">
    <RetryUntilSuccessful num_attempts="3" name="retry_nav">
      <Fallback name="nav_or_recover">
        <Sequence name="follow_path">
          <ComputePathToPose
              goal="{goal}"
              path="{path}"
              planner_id="GridBased"/>
          <FollowPath
              path="{path}"
              controller_id="FollowPath"/>
        </Sequence>

        <Sequence name="recover">
          <ClearEntireCostmap
              name="ClearLocal"
              service_name="local_costmap/clear_entirely_local_costmap"/>
          <Spin spin_dist="1.57"/>
          <ForceFailure name="recover_done_try_again"/>
        </Sequence>
      </Fallback>
    </RetryUntilSuccessful>
  </BehaviorTree>

  <TreeNodesModel>
    <Action ID="ComputePathToPose">
      <input_port name="goal"/>
      <output_port name="path"/>
      <input_port name="planner_id"/>
    </Action>
    <Action ID="FollowPath">
      <input_port name="path"/>
      <input_port name="controller_id"/>
    </Action>
  </TreeNodesModel>
</root>
```

说明：

- 恢复末尾 `ForceFailure`：让 Fallback 在「恢复执行完」后仍记为失败，从而触发外层 Retry 重新规划（一种常见写法）  
- 若希望「恢复成功即算导航成功」，不要 ForceFailure，改为恢复后再次嵌入规划子树  

## 4. 上层只发 goal 时的用法

任务节点不直接跑上述细树，而是：

```text
业务节点 → NavigateToPose(goal) Action → bt_navigator 内部树
```

此时本案例 XML 用于**定制 bt_navigator**；业务侧 XML 见 [[09-案例-多点巡检任务编排]]。

## 5. 黑板契约

| Key | 类型 | 谁写 | 谁读 |
|-----|------|------|------|
| `goal` | `PoseStamped` | 任务/Action 入口 | ComputePath |
| `path` | `Path` | ComputePath | FollowPath |
| `error_code` | int（可选） | Action 节点 | 日志 / HMI |

启动前：

```cpp
blackboard->set("goal", goal_msg);
```

## 6. 验收用例

| 用例 | 期望 |
|------|------|
| 空地目标可达 | 一次 Sequence 成功，无恢复 |
| 局部被临时障碍堵住后移开 | 走恢复 → Retry → 成功 |
| 目标在永久禁区 | 3 次后整树 FAILURE |
| 导航中取消 Action | 节点 `onHalted`，控制器停 |

## 7. 落地改法（开项目时）

1. 从 Nav2 自带 `navigate_to_pose_w_replanning_and_recovery.xml` 拷贝，再删减  
2. 先仿真（Gazebo / 官方 turtlebot 包）跑通，再上实车  
3. 把 `num_attempts`、Spin 角度、清哪张 costmap 做成参数或 XML 易改点
